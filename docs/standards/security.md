# 安全规范

本文档定义 UDBX 文件解析、SDK 和工具开发中的安全要求。

更高层的安全报告流程见根目录 `SECURITY.md`。

## 威胁模型

需要防范的输入包括：

- 损坏的 SQLite 文件。
- 恶意构造的 SQLite 文件。
- 长度字段异常的 GAIA blob。
- 类型字段异常的 CAD blob。
- 超大字段、超大 geometry、超大数据集。
- 路径穿越。
- 工具层误写输入文件。

## 二进制解析要求

解析前必须验证：

- 输入长度。
- magic byte。
- byte order。
- SRID 字段长度。
- MBR 字段长度。
- geometry type。
- 坐标数量。
- 结束标记。

禁止：

- 未检查长度直接读取。
- 信任文件内 count 直接分配巨大数组。
- 捕获异常后静默返回空几何。
- 为通过样本跳过边界检查。

## SQLite 访问要求

- 表名和字段名必须经过校验或安全转义。
- 值必须使用 prepared statement 或等价参数绑定。
- 批量写入必须使用事务。
- rows、statement、connection 必须明确关闭。

## 工具安全要求

工具默认应优先只读。

写操作必须：

- 明确告知用户。
- 不静默覆盖输入文件。
- 优先写入新文件或临时文件。
- 失败时不破坏原文件。

CLI 对高影响操作应有确认或清晰提示。

## 资源限制

工具和 SDK 应避免无界读取：

- preview 必须限制条数。
- 查询结果应支持分页。
- 导出应有进度和大小边界。
- 对超大 geometry 应考虑流式或分块处理。

## 错误处理

错误消息应包含足够上下文，但不能泄露：

- 凭据。
- 私有 token。
- 不必要的绝对路径。
- 私有样本内容。

## 安全测试

建议增加测试：

- 空文件。
- 非 SQLite 文件。
- 缺失系统表。
- 异常 `SmDatasetType`。
- 异常 `SmFieldType`。
- 截断 GAIA blob。
- 错误 byte order。
- 缺失结束标记。
- 超大长度字段。
