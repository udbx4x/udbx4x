# 风险登记

本文档记录 `udbx4x` 当前主要风险、影响、概率、缓解措施和负责人角色。

## 风险等级

- 高：可能阻碍发布、破坏兼容性或造成错误格式实现。
- 中：会影响开发效率、用户体验或工具成熟度。
- 低：影响局部质量或文档一致性。

## 优先处理顺序

1. 真实 SuperMap 主流 UDBX 样本 T3 准入、授权和兼容覆盖扩展。
2. 网络、三维模型等非最小合规类型的规范扩展设计。
3. 发布门禁和工具功能稳定性。
4. viewer、copilot、validator、converter 等工具生态稳定化。
5. 安全和畸形输入测试体系补齐。
6. 中英文文档与合规报告的持续同步。

## 风险关闭标准

风险关闭必须满足：

- 有对应文档或测试证明风险已缓解。
- 相关 SDK 或工具通过本地检查。
- 已知残余风险被记录。
- 若影响发布，`RELEASE.md` 或 changelog 已更新。
