# 安全策略

## 支持版本

每个活跃维护 SDK 的最新 minor version 接收安全修复。

活跃 SDK：

- `udbx4j`
- `udbx4ts`
- `udbx4go`

工具的安全支持跟随其依赖的 SDK 版本。

## 报告安全问题

请私下向 `udbx4x` GitHub 组织维护者报告安全问题。

在维护者有合理时间调查并发布修复前，不要公开漏洞利用细节、恶意样本文件或复现步骤。

## 安全范围

安全敏感区域包括：

- 损坏的 SQLite 数据库文件。
- 损坏的 GAIA geometry blob。
- 损坏的 CAD / GeoHeader blob。
- 二进制解析中的整数溢出和长度溢出。
- 越界读取和写入。
- 恶意文件触发的无限制内存分配。
- CLI 和桌面工具中的路径处理。
- 可能修改或破坏输入文件的写操作。
- 工具生态中的远程 planner 集成。

## 必要防御实践

- 读取前验证二进制长度。
- 验证字节序和 magic bytes。
- 解码坐标前验证 geometry type。
- 工具中的 preview、query、export 必须限制规模。
- 检查类工具优先使用只读访问。
- 写入测试必须使用临时文件。
- 不得静默覆盖输入文件。
- 可行时为畸形输入增加回归测试。

## 样本文件

不要在公开 issue 中附加私有或不可再分发的 `.udbx` 文件。

如果安全调查需要样本文件，请与维护者私下协作。

## 披露流程

1. 维护者确认收到报告。
2. 维护者复现问题并评估严重性。
3. 准备带回归测试的修复。
4. 发布修复版本。
5. 发布后记录漏洞说明。
